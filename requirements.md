# 陸上競技メニュー自動生成アプリ 要件定義書

**バージョン**: 1.0
**作成日**: 2026-03-13
**ステータス**: ドラフト

---

## 1. プロジェクト概要

### 1.1 アプリ名（仮）
Athletic Menu Generator

### 1.2 目的
陸上競技選手が、その日のコンディション（心理状態・筋肉痛・目標意識）を入力するだけで、AIが最適なトレーニングメニューを自動生成する Web アプリを提供する。

### 1.3 解決する課題
- 自分のコンディションに合ったメニューを自分で考えるのが難しい
- コーチがいない環境での練習計画が属人的になりがち
- 過去のトレーニング記録を活用できていない

---

## 2. ターゲットユーザー

| 項目 | 内容 |
|---|---|
| 対象 | 陸上競技選手（一般ユーザー） |
| スキルレベル | 初心者〜経験者まで幅広く |
| 種目 | 短距離 / 中長距離 / 跳躍 / 投擲（全種目対応） |

---

## 3. ユーザーストーリー

| # | ユーザーストーリー |
|---|---|
| US-01 | ユーザーとして、アカウントを作成・ログインして、自分のデータを管理したい |
| US-02 | ユーザーとして、自分の競技種目・年齢・性別・目標記録を登録したい |
| US-03 | ユーザーとして、その日のコンディションを入力して、AIにトレーニングメニューを生成してもらいたい |
| US-04 | ユーザーとして、生成されたメニューに詳細（セット数・回数・レスト・強度）が記載されていてほしい |
| US-05 | ユーザーとして、ウォームアップ・クールダウンもメニューに含めてほしい |
| US-06 | ユーザーとして、過去のトレーニング記録を振り返りたい |
| US-07 | ユーザーとして、大会スケジュールを登録してピーキングに対応したメニューを生成してほしい |

---

## 4. 機能要件

### 4.1 認証・アカウント機能（Must）

| 機能 | 詳細 |
|---|---|
| ユーザー登録 | メールアドレス + パスワードで新規登録 |
| ログイン / ログアウト | セッション管理、JWT 認証 |
| パスワードリセット | メールによるリセットリンク送信 |

### 4.2 プロフィール機能（Must）

| 項目 | 型 | 説明 |
|---|---|---|
| 競技種目 | Enum | 短距離 / 中長距離 / 跳躍 / 投擲 |
| 年齢 | Integer | メニュー強度の調整に使用 |
| 性別 | Enum | Male / Female / Other |
| 目標記録 | String | 例：100m 10秒台、マラソン3時間切りなど |
| 自己ベスト | String | 現在のベスト記録 |

### 4.3 コンディション入力機能（Must）

練習前に毎回入力する。

| 項目 | 入力形式 | 説明 |
|---|---|---|
| 心理・モチベーション | 5段階スライダー | やる気・集中力のレベル |
| 筋肉痛の状態 | 部位選択 + 3段階（なし/軽度/重度） | 脚・腕・体幹など部位ごとに選択 |
| 怪我の状態 | 部位選択 + フリーテキスト | 痛みのある部位と程度 |
| 目標意識 | 5段階スライダー | 今日の目標達成へのコミット度 |

### 4.4 メニュー生成機能（Must）

- **AI（Claude API）** を使用してトレーニングメニューを動的生成
- 入力情報（プロフィール + コンディション）をもとにプロンプトを構築してAPIへ送信
- 生成メニューに含まれる情報：

| 項目 | 内容 |
|---|---|
| ウォームアップ | 種目・時間・動的ストレッチ内容 |
| メインメニュー | トレーニング名・セット数・回数（距離）・レスト時間 |
| 負荷強度 | 強度 %（最大心拍数比など）または RPE（自覚的運動強度） |
| クールダウン | 静的ストレッチ・ジョグなどの内容 |
| 備考 | コンディションに応じた注意点・アドバイス |

### 4.5 トレーニング記録管理機能（Must）

| 機能 | 詳細 |
|---|---|
| 記録保存 | 生成したメニューとその日のコンディションを自動保存 |
| 記録一覧 | 日付順に過去のトレーニング記録を一覧表示 |
| 記録詳細 | 特定日のメニュー・コンディション・備考を表示 |

### 4.6 大会スケジュール機能（Nice to have）

| 機能 | 詳細 |
|---|---|
| 大会登録 | 大会名・日付を登録 |
| ピーキング対応 | 大会日からの残り日数をもとにAIがメニューの強度を自動調整 |

---

## 5. 非機能要件

| 項目 | 要件 |
|---|---|
| レスポンシブ対応 | スマートフォン・タブレット・PCで使用可能 |
| セキュリティ | パスワードのハッシュ化、JWT の適切な管理、XSS・CSRF 対策 |
| パフォーマンス | メニュー生成（AI応答）は最大30秒以内 |
| 可用性 | 特定の SLA は現時点では定めない |

## 7. データモデル（概要）

### User
```
id           String   (PK)
email        String   (unique)
passwordHash String
createdAt    DateTime
```

### Profile
```
id           String   (PK)
userId       String   (FK -> User)
event        Enum     (SHORT / MIDDLE_LONG / JUMP / THROW)
age          Int
gender       Enum     (MALE / FEMALE / OTHER)
targetRecord String
personalBest String
```

### TrainingSession
```
id              String   (PK)
userId          String   (FK -> User)
date            DateTime
motivation      Int      (1-5)
muscleSoreness  Json     (部位 -> 重度のマップ)
injuryStatus    Json     (部位 -> 詳細のマップ)
goalCommitment  Int      (1-5)
generatedMenu   String   (AIの生成結果テキスト or JSON)
createdAt       DateTime
```

### CompetitionEvent（Nice to have）
```
id        String   (PK)
userId    String   (FK -> User)
name      String
date      DateTime
createdAt DateTime
```

## 8. 技術スタック

| レイヤー | 技術 |
|---|---|
| フロントエンド | Next.js（App Router） |
| スタイリング | Tailwind CSS |
| バックエンド | Hono |
| ORM | Prisma |
| データベース | PostgreSQL |
| AI | Claude API（Anthropic） |
| 認証 | JWT（Hono middleware） |

### システム構成図（概要）

```
[Browser / Next.js]
        |
        | REST API (JSON)
        v
[Hono API Server]
    |         |
    |         v
    |    [Claude API]
    |    (メニュー生成)
    v
[Prisma ORM]
    |
    v
[PostgreSQL]
```

## 9. API エンドポイント（概要）

| メソッド | パス | 説明 |
|---|---|---|
